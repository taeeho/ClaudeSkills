현재 대화를 요약한 인수인계 문서를 작성해, 새로 시작한 에이전트가 이어서 작업할 수 있게 한다.

**저장 위치**: 사용자 OS의 임시 디렉토리. 현재 작업 폴더에는 저장하지 않는다.
- Windows: `$env:TEMP\handoff-<YYYYMMDD-HHmm>.md`
- macOS/Linux: `/tmp/handoff-<YYYYMMDD-HHmm>.md`

저장 후 절대경로를 사용자에게 출력한다.

## 문서에 포함할 것

1. **목적 (Purpose)** — 다음 세션이 무엇을 할 것인지 한두 문장
2. **현재 상태 (State)** — 지금까지 결정된 사항, 진행 중인 작업, 막혀 있는 지점
3. **참조 (References)** — 이미 다른 산출물(PRD/Plan/ADR/Issue/Commit/Diff/파일경로)에 있는 내용은 **복붙하지 말고 경로나 URL로 참조만** 한다
4. **다음 액션 (Next actions)** — 구체적이고 실행 가능한 단계
5. **Suggested skills** — 다음 세션에서 호출하면 좋은 스킬 목록 (예: `/bugfix`, `/plan-site`, `/qa-site`)
6. **주의사항 (Gotchas)** — 다음 에이전트가 모르고 들어가면 헤맬 만한 함정

## 금지

- API 키, 비밀번호, 개인정보(PII) 등은 자동으로 **redact**한다. 의심스러우면 `<REDACTED>`로 치환.
- 이미 코드/PR/이슈에 있는 내용을 다시 본문에 옮겨 적지 않는다 (경로 참조만).
- 본인 추측/의견은 명시적으로 "추정:" 접두어를 붙인다.

## 인자가 있을 경우

사용자가 인자(예: `/handoff "내일 API 통합 마무리"`)를 넘기면, 그 문장을 다음 세션의 초점으로 삼고 문서를 그쪽으로 기울인다.

## 언어

한국어 기본. 코드·식별자·에러 메시지·파일 경로는 원문 유지.
